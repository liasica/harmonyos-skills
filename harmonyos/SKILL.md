# HarmonyOS

Offline mirror of HarmonyOS NEXT developer documentation. Source: `developer.huawei.com/consumer/cn/doc/`. Every `.md` carries a YAML frontmatter with the original URL, breadcrumb, and `doc_updated_at`.

## Quick start

1. Locate the file via the index — never grep `references/` directly.

   ```bash
   rg -n "<keyword>" references/INDEX.md | head
   ```

2. Read the matched file. Its first 8 lines (frontmatter) tell you the original URL and last-modified date.

3. When the user needs a citation, return the `url` field from frontmatter — that is the canonical Huawei documentation link.

## Categories

| Path | Contents |
|---|---|
| `references/harmonyos-releases/` | Version notes, API levels, SDK changelogs |
| `references/harmonyos-guides/` | Application framework, ArkTS, ArkUI, media, network — usage guides |
| `references/harmonyos-references/` | API reference (`@ohos.*` modules, components, error codes) |
| `references/best-practices/` | Performance, compatibility, energy — official best practices |
| `references/harmonyos-faqs/` | Frequently asked questions |

Each category has its own `INDEX.md` listing titles + relative paths, useful for narrowing scope:

```bash
rg -n "UIAbility" references/harmonyos-references/INDEX.md
rg -n "@ohos\.app\.ability" references/harmonyos-references/INDEX.md
```

## Internal links are local

Links inside any `.md` that originally pointed to the 5 whitelisted root URLs have been rewritten to relative paths into this repository. Links pointing outside the whitelist remain absolute HTTPs.

## Workflow

1. Clarify the user's request — feature lookup, API signature, error code, version diff, or best practice?
2. Match keywords against `references/INDEX.md` (or a category-scoped `INDEX.md` when the category is obvious).
3. `Read` only the matched `.md`. Do not read sibling files unless explicitly required.
4. Cite the `url` from frontmatter when answering.
5. If the local file is older than the user's project requires (compare `doc_updated_at`), or if no local match exists for a HarmonyOS topic, fetch the live page from `developer.huawei.com` and disclose the fallback.

## Quality rules

- Trust `references/` over training-time memory for `@ohos.*` API signatures, parameter types, and return values.
- For ArkUI examples, prefer declarative syntax (`@Entry`, `@Component`, `build()`) unless the documentation is explicitly about NDK or a system service.
- Do not modify any file under `references/` — they are scraped artifacts and will be overwritten on the next sync.
- Do not paste frontmatter fields (`content_hash`, `scraped_at`, `category`) into generated code; they are scraper metadata.
- When two pages disagree (e.g. an old guide vs. a new release note), surface the conflict and cite both.

## Sync

Documentation is regenerated by the `scraper/` sub-project. To refresh manually:

```bash
cd scraper && uv run python -m scripts.sync
```

Resumes from `scraper/data/manifest.json` if interrupted. Detailed usage is in [`../README.md`](../README.md).
