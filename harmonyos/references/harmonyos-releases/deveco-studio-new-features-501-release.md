## DevEco Studio 5.0.1 Release（5.0.5.306）新增和增强特性

### 新增特性

* DevEco Studio支持开发API 13工程。
* DevEco Profiler Frame模板新增Lost Frames和Hitch Time泳道，用于识别和优化卡顿和丢帧现象。具体请参考[Frame分析](../harmonyos-guides/ide-insight-session-frame.md)。
* hvigor-config.json5中properties下新增ohos.arkCompile.noEmitJs字段，用于指定ArkTS编译过程中是否生成js中间产物，不生成js中间产物可以降低编译过程的峰值内存，加快编译速度。具体请参考[hvigor-config.json5](../harmonyos-guides/ide-hvigor-set-options.md)。
* hvigor新增以下API。具体可参考[插件上下文](../harmonyos-guides/ide-build-expanding-context.md)和[基础构建能力](../harmonyos-guides/ide-hvigor-api.md)。
  + getOverrides：获取工程下oh-package.json5中配置的overrides字段。
  + setOverrides：设置工程下oh-package.json5中的overrides字段。
  + setProperty：设置hvigor-config.json5配置文件中properties对象指定key值的value值。
* 新增支持HWASan检测。具体请参考[HWASan检测](../harmonyos-guides/ide-hwasan.md)。
* 支持在DevEco Studio内上传应用软件包。具体请参考[上传软件包](../harmonyos-guides/ide-publish-app.md#section97874500234)。
* 应用与服务体检新增一条快速性能检测规则：避免序列化反序列化耗时长。具体请参考[规则总览](../harmonyos-guides/ide-app-analyzer-all-rules.md)。
* ohpm新增支持resolve\_conflict\_strict严格模式依赖冲突处理功能。具体请参考[ohpmrc](../harmonyos-guides/ide-ohpmrc.md)。

### 增强特性

* 意图框架新增支持多种垂域类型。具体请参考[创建意图框架](../harmonyos-guides/ide-insight-intent.md)。
* DevEco Profiler工具录制提供可视化配置。具体请参考[数据区](../harmonyos-guides/ide-profiler-data.md)。
* hvigorw命令行工具支持--max-old-space-size参数，用于设置守护进程内存大小。具体可参考[设置守护进程内存](../harmonyos-guides/ide-hvigor-daemon.md#section327617383145)和[hvigorw](../harmonyos-guides/ide-hvigor-commandline.md#section101034488138)。
* AppAnalyzer检测报告支持与Profiler工具使用时间戳联动，点击时间戳可以打开Profiler并定位到问题发生的时间点。具体可参考[应用与元服务体检](../harmonyos-guides/ide-app-analyzer.md)。
* Launch分析支持手动冷启动场景。具体请参考[冷启动分析：Launch分析](../harmonyos-guides/ide-insight-session-launch.md)。

### 变更特性

* Codelinter部分性能规则所属规则集变更。具体请参考[规则变更说明](../harmonyos-guides/ide-codelinter-rules-change.md)。

新建工程或模块时，resources下默认不再创建en\_US和zh\_CN目录。如需实现多语言资源配置，可创建相应限定词目录。具体请参考[资源目录](../harmonyos-guides/resource-categories-and-access.md#资源目录)。
